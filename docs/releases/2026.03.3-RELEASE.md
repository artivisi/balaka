# Release 2026.03.3

## Highlights

Product rebrand: **Aplikasi Akunting** is now **Balaka**. New name, logo, favicon, domain, and GitHub repo. No schema changes, no API breaking changes.

## What's New

### Rebrand

- Product renamed from "Aplikasi Akunting" to "Balaka"
- New B mark logo (deep indigo #2E2D8E) with Plus Jakarta Sans wordmark
- New favicon (Balaka B mark)
- Default theme switched from `artivisi` to `balaka`
- Login page: product heading "Balaka" + "Powered by Balaka" footer
- Page title suffix changed from "Akunting" to "Balaka"
- OpenAPI title: "Balaka API"
- Data export manifest: "Application: Balaka"
- Telegram bot messages updated
- User manual HTML title and footer updated

### Infrastructure

- GitHub repo renamed: `artivisi/aplikasi-akunting` → `artivisi/balaka` (old URL auto-redirects)
- Artivisi domain: `akunting.artivisi.id` → `balaka.artivisi.id`
- DNS records created: `balaka.artivisi.id`, `balaka.id`, `www.balaka.id`
- Print template fallback logo now uses theme path (`${theme.logoPath}`) instead of hardcoded artivisi logo

### Documentation

- User manual updated: GitHub URLs, example domains, product name references
- README, CLAUDE.md updated with new repo URL and domain
- Deploy repo README and scripts updated

## Bug Fixes

- None

## Breaking Changes

- None. The API, schema, and all endpoints are unchanged.
- Clients using `app.theme.name=artivisi` continue to work — the artivisi theme remains in the codebase.

## Migration Guide

### For existing deployments

1. Update git remote on server: `git remote set-url origin https://github.com/artivisi/balaka.git`
2. Update DNS: add A record for new domain pointing to the same server IP
3. Deploy with Ansible to set up nginx vhost + SSL for new domain
4. Add nginx redirect from old domain to new domain
5. Update Telegram webhook URL if bot is enabled

### Theme override

If your deployment overrides `app.theme.name` via Ansible vars (e.g., `app_theme_name: artivisi`), no change needed — client themes are unaffected. The new `balaka` default only applies when no override is set.
